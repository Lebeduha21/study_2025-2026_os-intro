# Актульность

- Сети объединяют **разнородное оборудование, различные ОС и программы**.
- Успешное взаимодействие → принятие общепринятых стандартов:
  - **OSI** (международный стандарт)
  - **IETF** (набор стандартов Internet)
- Понимание архитектур необходимо для администрирования, настройки и диагностики сетевой инфраструктуры, особенно в UNIX-подобных системах.

# Распределенные системы — определение и свойства

**Распределенная система** — набор независимых компьютеров, представляющихся пользователям единой объединенной системой.

| Свойство | Характеристика |
|----------|----------------|
| **Прозрачность** | Скрытие различий между компьютерами и способов связи |
| **Открытость** | Использование стандартизованных протоколов и интерфейсов |
| **Масштабируемость** | Расширение по размеру, географии и административному устройству |

# Логические слои и типы архитектур

**Логические слои распределенных систем:**

1. **Презентационный слой** — взаимодействие с пользователем
2. **Слой прикладной логики** — обработка данных (бизнес-логика)
3. **Слой управления ресурсами** — доступ к данным и внешним системам

**Типы по количеству ярусов:**

- Одноярусные (монолит)
- Двухъярусные (клиент-сервер)
- Трехъярусные (с выделением middleware)
- Многоярусные (объединение разнородных систем)

# Способы взаимодействия в распределенных системах

| Тип | Описание | Пример |
|-----|----------|--------|
| **Синхронное** (блокирующее) | Клиент ждет ответа от сервера | Классический RPC |
| **Асинхронное** (неблокирующее) | Клиент продолжает работу после запроса | Электронная почта, очереди сообщений |

**Преимущество асинхронного взаимодействия** - слабая связанность компонентов, устойчивость к сбоям.

# Локальные сети (LAN) — определение и топологии

**LAN (Local Area Network)** — коммуникационная система в пределах здания/территории с высокоскоростными цифровыми каналами связи.

Топологии LAN:

- Шина — все компьютеры подключены к общему кабелю
- Звезда — каждый компьютер подключен к центральному устройству
- Кольцо — компьютеры соединены последовательно в замкнутую цепь

# Модель OSI (7 уровней)

| Уровень | Назначение |
|---------|------------|
| 7. Прикладной | Управление вычислительными процессами, доступом к внешним устройствам |
| 6. Представительный | Доступ к файлам данных, преобразование данных в требуемый формат |
| 5. Сеансовый | Установление и завершение логических соединений |
| 4. Транспортный | Фрагментация и сборка сообщений, передача файлов |
| 3. Сетевой | Маршрутизация пакетов, управление потоками данных |
| 2. Канальный | Управление передачей кадров, контроль ошибок |
| 1. Физический | Управление физическими соединениями, тактирование |

# Модель IEEE 802 и стек TCP/IP

**Модель IEEE 802** (модификация OSI для LAN):

- Канальный уровень разделен на два подуровня:
  - **LLC (Logical Link Control)** — универсальный для всех LAN
  - **MAC (Medium Access Control)** — определяет топологию и метод доступа

**Стек TCP/IP** (наиболее «оригинальный» для UNIX):

- IP — сетевой уровень (продвижение пакетов)
- TCP — надежная доставка с установлением соединения
- UDP — быстрая, ненадежная доставка
- ICMP — управление сообщениями об ошибках

# Сетевой интерфейс и конфигурация IP-сетей

**Сетевой интерфейс** — абстракция, связывающая канальный и сетевой уровни.

**Характеристики интерфейса:**

- Уникальное имя (eth0 — для Ethernet, lo — loopback)
- IP-адрес и маска сети
- Аппаратный адрес (MAC-адрес)

**IP-адресация:**

- IPv4: 4 байта, запись в десятичной системе (192.168.1.5)
- Loopback (lo): 127.0.0.1 (связь компьютера с самим собой)

**Маршрутизация:**

- IP-адрес = адрес подсети + адрес узла
- Маска подсети определяет границу между ними

# Служба DNS

**DNS (Domain Name Service)** — распределенная сетевая база данных.

**Назначение:** преобразование доменных имен в IP-адреса.

**Иерархическая структура:**

- Корневой домен (без имени)
- Домены верхнего уровня (com, edu, gov, ru, us и др.)
- Домены второго уровня (yandex.ru, rudn.ru и т.д.)

**Принцип работы:** клиент обращается к DNS-серверу, который находит соответствующий IP-адрес в распределенной базе.

# Заключение

Архитектуры распределенных систем и LAN основаны на OSI/TCP/IP — UNIX реализовал их в своей сетевой подсистеме, что стало основой Internet. Понимание этих принципов необходимо для администрирования сетей.
